# Development Guide: Клёво

## Quick Start

```bash
# 1. Clone and install
git clone <repo> && cd klevo
pnpm install

# 2. Configure environment
cp .env.example .env
# Fill in: CLAUDE_API_KEY, YANDEX_GPT_API_KEY, SUPABASE_URL, SUPABASE_ANON_KEY, etc.

# 3. Bootstrap full project from docs
/start

# 4. Or start services manually
docker compose up -d
# Then visit http://localhost:3000
```

## Full Lifecycle

```
Setup → Develop → Test → Deploy
  /start    /feature  /test  /deploy
```

## Command Reference

| Command | Purpose | When to Use |
|---------|---------|-------------|
| `/start` | Bootstrap monorepo from SPARC docs | First time project setup |
| `/feature [name]` | Full 4-phase lifecycle (plan→validate→implement→review) | New features |
| `/plan [task]` | Lightweight implementation plan | Simple tasks, hotfixes |
| `/go [feature]` | Auto-select pipeline by complexity | When unsure which pipeline |
| `/run` | Implement full MVP autonomously | Autonomous coding session |
| `/run all` | Implement all features | Full project automation |
| `/test` | Run test suite | After any change |
| `/test [feature]` | Run Gherkin scenarios for feature | Feature-specific testing |
| `/deploy` | Deploy to VPS with pre-flight checks | Production deployments |
| `/next` | Show sprint progress + top 3 actions | Start of each session |
| `/next [id]` | Mark feature done, unblock dependencies | After completing feature |
| `/myinsights [title]` | Capture development insight | After tricky bug fix |
| `/docs` | Generate bilingual documentation | Release preparation |

## Monorepo Structure

```

├── apps/
│   ├── web/           Next.js 15 (frontend + BFF)
│   └── ai-service/    Python FastAPI (AI orchestration)
├── packages/
│   ├── db/            Supabase schema + RLS + migrations
│   ├── ui/            Shared React components
│   └── types/         Shared TypeScript types
├── services/
│   ├── nginx/         Reverse proxy (SSL termination)
│   └── redis/         Rate limiting + session cache
├── docs/
│   ├── *.md           SPARC documentation (source of truth)

│   └── plans/         Implementation plans (generated by /plan)

    ├── agents/        @planner, @code-reviewer, @architect

    ├── rules/         Coding standards, security, git workflow
    ├── skills/        sparc-prd-mini, testing-patterns, etc.

```

```

/docs rus      → Russian documentation only

```

Command hierarchy: `/run` → `/start` → `/next` → `/go` → `/plan` | `/feature`

## Troubleshooting

**Check insights first:**
```bash
grep -i "YOUR_ERROR" myinsights/1nsights.md
```

**Common issues:**
- T-Bank CSV encoding → see `coding-style.md` Russian-Market Specifics section
- Claude API from RU VPS → configure `PROXY_API_URL` in `.env`
- Supabase RLS blocking queries → verify user is authenticated, check RLS policies in `packages/db/schema/`
- Redis rate limit 429 → user hit 10 req/min limit, wait 60s or check `rate_limiter.py`

## Security Checklist (before every PR)

- [ ] No API keys in code or git diff (`git diff --cached | grep -E "sk-ant|api_key"`)
- [ ] Zod validation on new API routes
- [ ] RLS policy added for new DB tables
- [ ] JWT httpOnly cookies used (not localStorage)
- [ ] CSV raw file not stored on disk
- [ ] Rate limiting applied to new AI endpoints
