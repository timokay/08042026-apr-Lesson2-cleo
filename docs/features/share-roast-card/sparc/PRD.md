# PRD: Share Roast Card

## Проблема

Пользователь получил ростер и хочет им похвастаться. Нет механики виральности без шеринга.

## Цель

Дать возможность поделиться ростером как публичной страницей. Ссылка работает без логина. OG-превью для Telegram/VK/WhatsApp.

## MoSCoW

### Must Have
- Публичная страница `/share/[token]` без авторизации
- Уникальный URL-safe токен (8 байт, base64url)
- OG meta tags для превью в мессенджерах (title, description, image)
- Кнопка копирования ссылки

### Should Have
- Reuse существующего токена (не генерировать новый при повторном шеринге)
- is_public=true флаг для RLS

### Could Have
- Кастомная OG image через Vercel/SATORI
- Счётчик просмотров

### Won't Have (v1)
- Редактируемый публичный профиль
- Встроенный widget для других сайтов

## Метрики

| Метрика | Цель |
|---------|------|
| Share rate (от пользователей завершивших ростер) | ≥ 25% |
| Klick rate по shared link | ≥ 10% |
| Новые регистрации через share | tracked |

## User Flow

```
/roast → Done состояние
  → "Поделиться" кнопка
  → POST /api/share { roast_id }
  → { share_url: "https://klevo.app/share/abc123" }
  → Clipboard copy + "Скопировано!" feedback
  → Получатель открывает /share/abc123
  → Видит ростер без логина (публичная страница)
```
