# PRD: Parasite Scanner

## Проблема

Пользователи платят за подписки, о которых забыли. Средний россиянин 18–30 лет имеет 4–7 активных подписок и знает только о половине (исследование Яндекс.Плюс, 2024).

## Цель

Автоматически обнаружить recurring-паттерны в транзакциях и показать список "паразитных" подписок с суммой в месяц. Viral hook: "Ты тратишь ₽2,847 в месяц на подписки, которыми не пользуешься".

## Пользователи

Молодёжь 18–30 лет, часто подписываются на триалы и забывают отменить.

## MoSCoW

### Must Have
- Обнаружение recurring-паттернов (std deviation < 5 дней)
- Показ top-20 подписок, отсортированных по сумме/мес
- 3 уровня confidence: HIGH (из KNOWN_SUBSCRIPTIONS), MEDIUM (паттерн), LOW
- Сумма "ежемесячных потерь" вверху списка
- Кнопки "оставить" / "отменить" (UI)

### Should Have
- Обнаружение неактивных подписок (> 45 дней без списания)
- KNOWN_SUBSCRIPTIONS список (Netflix, Spotify, Яндекс Плюс, etc.)

### Could Have
- Ссылка для отмены подписки (если известен URL)
- Уведомления о новых подписках

### Won't Have (v1)
- Автоматическая отмена подписок
- Интеграция с банком для блокировки

## Метрики успеха

| Метрика | Цель |
|---------|------|
| Precision (реальные подписки / найденные) | ≥ 80% |
| Recall (найденные / реальные) | ≥ 70% |
| HIGH confidence accuracy | ≥ 95% |
| Время выполнения (1000 транзакций) | < 500мс |

## User Flow

```
Пользователь загружает CSV
  → find_parasites() вызывается автоматически в /analyze
  → SubscriptionList показывается на dashboard рядом с PieChart
  → Пользователь видит "Паразиты: ₽2,847/мес" → 😱
  → Нажимает "Поджарь!" → получает ростер с упоминанием подписок
```
