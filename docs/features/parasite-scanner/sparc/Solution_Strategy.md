# Solution Strategy: Parasite Scanner

## Ключевые технические решения

### 1. Statistical pattern detection (не ML)

**Решение:** std_dev < 5 дней + 7 ≤ avg_interval ≤ 35 дней

**Обоснование:** 
- ML-подход требует labeled training data (нет для РФ рынка)
- Статистика работает детерминированно — легко тестировать
- precision ~80%+ для реальных подписок
- Нет latency от внешнего API

**Альтернатива:** rule-based keyword matching только — низкий recall для неизвестных сервисов.

### 2. KNOWN_SUBSCRIPTIONS seed list для HIGH confidence

**Решение:** Hardcoded set ~30 известных сервисов + statistical overlap = HIGH.

**Обоснование:** Пользователь больше доверяет HIGH confidence кнопке "отменить". Ложные HIGH = потеря доверия. Только для явно известных: Netflix, Spotify, Яндекс, VK, Apple.

### 3. avg_interval нормализация к 30 дням

**Решение:** `monthly = avg_amount * (30 / avg_interval)`

**Обоснование:** Еженедельные подписки (avg=7) показываются как месячная стоимость. Позволяет сравнивать подписки с разной периодичностью.

**Риск:** Если avg_interval=14 и сумма=149₽ → monthly=320₽. Фактически 2× в месяц. Корректно.

### 4. is_active threshold 45 дней

**Решение:** (today - last_charge) > 45 дней → is_active=False

**Обоснование:** Большинство ежемесячных подписок списываются каждые 28–31 день. 45 дней = пропущен один цикл + небольшой буфер. Если неактивна — всё равно показываем (пользователь мог пропустить).

### 5. top-20 ограничение

**Решение:** Возвращаем только 20 подписок, отсортированных по monthly cost DESC.

**Обоснование:** UX — больше 20 = информационная перегрузка. Самые дорогие первыми = максимальная ценность.
