# PRD: Roast Mode (AI Финансовый Ростер)

## Проблема

Финансовые советники скучные и формальные. Молодёжь их игнорирует. Нужен способ заставить пользователя смеяться над своими расходами — и одновременно задуматься.

## Цель

Сгенерировать персонализированный юмористический "ростер расходов" с конкретными советами. Это главная WOW-фича и вирусный механизм: "Поделиться ростером".

## Пользователи

Молодёжь 18–30 лет, хотят развлечься и получить совет. Ключевой контент для шеринга.

## MoSCoW

### Must Have
- Streaming SSE (пользователь видит как появляется текст)
- Персонализация по категориям и подпискам
- Ответ строго на русском (retry если AI ответил на English)
- Claude 3.5 Sonnet как primary (через proxyapi.ru)
- YandexGPT 3 как fallback при Claude timeout/ошибке
- Rate limiting: 10 запросов/мин/пользователь (Redis)
- Free plan: 1 ростер в месяц

### Should Have
- Generic template fallback при отказе обоих AI
- Минимальный порог: < 10 транзакций → "недостаточно данных"

### Could Have
- Выбор тона (мягкий / жёсткий)
- Ростер по категории (только Еда, только Подписки)

### Won't Have (v1)
- Voice roast (audio)
- Scheduled roasts (каждую неделю)

## Метрики успеха

| Метрика | Цель |
|---------|------|
| Время первого токена (TTFT) | < 2.5с |
| Fallback rate (Claude → YandexGPT) | < 10% |
| Completion rate (пользователь дочитал) | ≥ 80% |
| Share rate (кнопка "поделиться") | ≥ 25% |
| Язык ответа = русский | 100% |

## User Flow

```
Пользователь на /dashboard нажимает "Поджарь!"
  → /roast page, состояние: streaming
  → SSE stream: текст появляется по токену
  → Done: кнопка "Поделиться" + кнопка "Ещё раз"
  → [или] Rate limited: "Вернись завтра" + CTA на Plus
  → [или] Insufficient data: "Загрузи больше транзакций"
```
