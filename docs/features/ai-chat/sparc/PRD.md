# PRD: AI Chat

**Feature ID:** ai-chat
**Sprint:** v1.0 (Months 3-4)
**Status:** Planning
**Depends on:** auth, roast-mode

---

## Problem Statement

Пользователи Клёво получают одноразовый ростер (roast-mode), но не могут задать уточняющие вопросы: "почему так много на еде?", "что сделать чтобы сэкономить?", "покажи тренд за 3 месяца". Нет диалога — нет глубины.

## Solution

Conversational AI-советник с характером: знает транзакции пользователя, отвечает с юмором и конкретикой, поддерживает контекст в рамках сессии.

## Target Users

Российская молодёжь 18-30 лет, уже загрузившие CSV и получившие ростер. Чат — следующий шаг вовлечения после "подгореть".

## Feature Scope (MoSCoW)

### Must Have
- Текстовый чат с AI о личных финансах
- Контекст транзакций в промпте (суммарно по категориям + паразиты)
- SSE streaming ответов (как в roast-mode)
- Auth required
- История сообщений в рамках сессии (in-memory)
- Claude primary, YandexGPT fallback

### Should Have
- Персистентная история чата в DB (chat_messages table)
- Предустановленные вопросы-подсказки (quick replies)
- "Характер" (тон: дружелюбная честность, без занудства)

### Could Have
- Голосовой ввод (Web Speech API)
- Экспорт чата в PDF

### Won't Have (v1)
- Мультиагентность
- Интеграция с банковскими API напрямую
- Push-нотификации по финансовым событиям

## Success Metrics

| Метрика | Цель | Срок |
|---------|------|------|
| DAU chat / DAU total | ≥ 30% | Через 2 недели после релиза |
| Avg messages per session | ≥ 4 | Через 1 месяц |
| Plus conversion from chat | ≥ 5% | Через 1 месяц |
| First response < 2.5s (first token) | ≥ 95% | С релиза |

## User Flow

1. Пользователь открывает `/chat`
2. Видит приветствие + 3 quick reply кнопки
3. Вводит вопрос или нажимает quick reply
4. AI отвечает стримингом (SSE)
5. История накапливается в рамках сессии
6. Free план: 10 сообщений/день, Plus: безлимит
