# PRD: CSV Upload

## Проблема

Пользователь хочет загрузить выписку из банка и сразу получить аналитику расходов. Без импорта данных приложение не работает.

## Цель

Принять CSV-файл из Т-Банка (и других РФ-банков), извлечь транзакции расходов, сохранить в БД и вернуть категоризированную сводку.

## Пользователи

Российская молодёжь 18–30 лет, получают выписку в личном кабинете банка.

## MoSCoW

### Must Have
- Загрузка CSV файла (drag-and-drop + click)
- Поддержка Т-Банк CSV (UTF-8 и cp1251)
- Фильтрация: только расходы (amount < 0)
- Ограничение 1000 строк, предупреждение при усечении
- Дедупликация по (date, amount, description)
- Сохранение Transaction[] в Supabase
- Лимит файла 10 МБ (Next.js) / 5 МБ (AI service)

### Should Have
- Поддержка Сбербанк и Альфа-Банк CSV
- Generic fuzzy-формат для неизвестных банков
- Понятные сообщения об ошибках на русском

### Could Have
- Банки: Тинькофф Бизнес, Озон Банк
- Batch-загрузка нескольких файлов

### Won't Have (v1)
- PDF-выписки
- Прямая интеграция с банковским API (Open Banking)

## Метрики успеха

| Метрика | Цель |
|---------|------|
| Успешный парсинг Т-Банк CSV | ≥ 95% |
| Время парсинга 1000 строк | < 2 секунды |
| Error rate по понятным ошибкам | 100% — всегда русский текст |
| Доля пользователей, прошедших загрузку | ≥ 70% первого посещения |

## User Flow

```
Пользователь открывает /dashboard
  → видит зону загрузки CsvUploadZone
  → перетаскивает / выбирает файл
  → идёт прогресс-спиннер
  → транзакции сохранены → показывается CategoryPieChart + SubscriptionList
  → [или] ошибка → понятный текст на русском
```
