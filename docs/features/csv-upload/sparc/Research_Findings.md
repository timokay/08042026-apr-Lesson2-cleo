# Research Findings: CSV Upload

## Форматы CSV российских банков

### Т-Банк (Тинькофф)
- Кодировка: UTF-8 (веб), cp1251 (мобильное приложение на старых Android)
- Разделитель: `;`
- Ключевые колонки: `Дата операции`, `Сумма операции`, `Описание`, `MCC`
- Формат даты: `DD.MM.YYYY` и `DD.MM.YYYY HH:MM:SS`
- Особенность: знак минус для расходов, плюс для доходов

### Сбербанк
- Кодировка: UTF-8
- Разделитель: `;`
- Ключевые колонки: `Дата проводки`, `Сумма`, `Описание`, `Назначение`
- Особенность: суммы с буквой валюты ("RUR"), требует strip

### Альфа-Банк
- Кодировка: UTF-8
- Разделитель: `,`
- Ключевые колонки: `Дата транзакции`, `Сумма`, `Категория`
- Особенность: иногда добавляет BOM (UTF-8 with BOM)

## Конкуренты и их подходы

| Продукт | Подход | Поддержка РФ банков |
|---------|--------|---------------------|
| Дзен-мани | CSV + Open Banking | Широкая |
| CoinKeeper | Ручной ввод | Нет CSV |
| Wallet by BudgetBakers | CSV import | Частичная |
| Cleo (US) | Plaid API | Нет (США) |

**Вывод:** Cleo использует Plaid (банковский API), в РФ нет аналога. CSV — единственный реалистичный вариант для v1.

## Технические находки

- `\u00a0` (неразрывный пробел) встречается в суммах из Т-Банка — требует очистки в `parse_amount`
- Формат `-1 234,56` (пробел как разделитель тысяч, запятая как десятичная) специфичен для РФ
- cp1251 encoding → `errors="replace"` лучше, чем `errors="strict"` (не падаем на одном плохом символе)
- csv.DictReader нечувствителен к регистру заголовков ТОЛЬКО при ручной нормализации

## Риски

- Банки меняют формат CSV без предупреждения (Т-Банк менял 2 раза за 2023 год)
- Mitigation: generic fuzzy-parser + версионирование форматов
