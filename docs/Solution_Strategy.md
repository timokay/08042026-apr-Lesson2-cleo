# Solution Strategy: Клёво
**Версия:** 1.0 | **Дата:** 2026-04-08

---

## Problem Statement (SCQA)

**Situation:** Российская молодёжь 18-30 лет использует мобильные банки каждый день, но большинство не управляет финансами осознанно — деньги "просто заканчиваются".

**Complication:** Существующие инструменты (Дзен-мани, CoinKeeper) требуют дисциплины ручного ввода и не мотивируют. Банковские AI скучны. Western fintech (Cleo) недоступен в РФ. Молодёжь знает о проблеме, но не имеет инструмента, которым хочется пользоваться.

**Question:** Как создать AI-финансового советника, которого молодёжь будет использовать с удовольствием — и который приведёт к реальным изменениям поведения?

**Answer:** Продукт с "характером" — юмористический, честный, моментально ценный. Паразит-детектор как hook (немедленная ценность) + Ростер как core loop (вирусность + вовлечение) + Цели как retention (LTV).

## First Principles Analysis

**Фундаментальные истины:**
1. Люди меняют поведение под влиянием эмоций, не информации
2. Юмор снижает психологическое сопротивление к неудобной правде о деньгах
3. Немедленная ценность важнее долгосрочных обещаний для acquisition
4. Вирусный контент = органический рост = низкий CAC
5. В РФ нет персонализированного AI-финансового советника с характером
6. Данные о тратах = самые чувствительные ПДн → нужно максимальное доверие

**Выводы для продукта:**
- Юмор как механика изменения поведения (не развлечение само по себе)
- Немедленный результат (60 сек до первого ростера)
- Шеринг = product-led growth инструмент
- Локальный хостинг (РФ) = доверие к хранению ПДн

## Root Cause Analysis (5 Whys)

**Проблема:** Молодёжь не управляет деньгами.

1. **Why?** → Не используют инструменты учёта
2. **Why?** → Инструменты неудобны/скучны (нужно вводить каждую трату)
3. **Why?** → Нет автоматизации + нет мотивации использовать
4. **Why?** → Нет интеграции с банками + нет геймификации/развлечения
5. **Why (Root Cause)?** → Инструменты созданы для тех, кто уже мотивирован — не для тех, кому нужна мотивация

**Вывод:** Нужен продукт, который создаёт мотивацию, а не требует её наличия.

## Game Theory Analysis

**Игроки:**
| Игрок | Интерес | Nash Strategy |
|-------|---------|--------------|
| Пользователь (Максим) | Понять траты без усилий, посмеяться, не чувствовать себя дураком | Использовать бесплатно, делиться |
| Клёво (бизнес) | CAC < 150₽, LTV > 3000₽, K > 1 | Вирусный бесплатный tier, платный апгрейд |
| Т-Банк / Сбер | Удержать клиентов в экосистеме | Скопировать фичу (угроза) |
| Дзен-мани / CoinKeeper | Конкурировать за аудиторию | Добавить AI (медленнее нас) |

**Nash Equilibrium:** Клёво выигрывает при скорости (First Mover в нише "personality fintech РФ") + вирусности (конкуренты не могут быстро скопировать personality).

**Second-Order Effects:**
- Если успешны → Т-Банк или Сбер купит или скопирует → план B: patent-защита personality system + партнёрство с банком
- Если вирусность работает → CAC → 0 → unit economics сходятся быстро
- Если ростер обижает → репутационный риск → нужен А/Б тест тональности

## TRIZ Contradictions Resolved

| Противоречие | TRIZ Принцип | Решение |
|--------------|-------------|---------|
| Хотим быть честными (говорить неудобное) vs. Не хотим обидеть | #35: Transformation of Properties | Юмор = "мягкая честность" — неудобная правда в смешной форме |
| Нужно много данных для точного анализа vs. Пользователь не хочет давать доступ к банку | #1: Segmentation | MVP: CSV upload (минимальный доступ), потом API (больше данных) |
| Хотим вирусность vs. Финансовые данные приватны | #40: Composite Materials | Делимся не данными, а "ростером" — обезличенным мемом |
| Нужен бесплатный tier для роста vs. Нужна выручка для выживания | #25: Self-service | Бесплатный ростер = реклама продукта = viral acquisition |
| LLM API дорого vs. нужны AI ответы в реальном времени | #35 | Кэширование похожих запросов + промпты с меньшим количеством токенов |

## Recommended Approach

### Стратегия GTM (Go-to-Market)

**Phase 1 (Месяц 1-2): Seed Virality**
- Запуск в студенческих чатах Telegram (>50 чатов, 200K+ аудитория)
- 10 VK-постов с реальными ростерами (согласие пользователей с фокус-группы)
- TikTok: 3 видео "AI поджаривает мои расходы"
- Цель: 1,000 первых пользователей бесплатно

**Phase 2 (Месяц 3-4): Product-Led Growth**
- Механика шеринга ростера — каждый пользователь = потенциальный реферер
- Реферальная программа: пригласи 3 → месяц Plus бесплатно
- Цель: K > 1.2, CAC < 150₽

**Phase 3 (Месяц 5-6): Monetization Optimization**
- A/B тест paywall timing (сразу после ростера vs. после 3 дней)
- Upsell к Plus через push-notifications (финансовые инсайты)
- Цель: Free→Plus > 12%, MRR > 3M₽

### Технический MVP Path
```
Week 1-2: Инфраструктура
  → Monorepo setup, Docker Compose, Supabase
  
Week 3-4: Core features
  → CSV parser, transaction categorizer, Топ-5 дашборд
  
Week 5-6: AI Roaster + Parasite Detector
  → Prompt engineering для ростера
  → Rule-based + AI паразит-сканер
  → Sharing карточка генератор
```

## Risk Assessment

| Риск | Вероятность | Влияние | Митигация |
|------|:-----------:|:-------:|-----------|
| Ростер культурно не попадает (обидный, а не смешной) | 35% | Высокое | Фокус-группа 20 чел. до запуска; Soft tone тест |
| LLM API недоступен из РФ | 60% | Критическое | API прокси (Proxyapi.ru или VPN на VPS) + YandexGPT fallback |
| ФЗ-152 нарушения | 25% | Критическое | VPS в РФ с первого дня; юридическая консультация до запуска |
| Конкурент копирует раньше выхода | 20% | Среднее | Скорость (MVP за 6 нед.) + strong branding |
| CSV форматы банков меняются | 50% | Низкое | Гибкий парсер + manual fallback |
| Retention низкий без банковской интеграции | 40% | Высокое | Telegram-бот как альтернативный канал напоминаний |
